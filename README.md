# Основные возможности

## Для пользователей:
- **Регистрация и аутентификация (JWT)**
    - Настроено получение access и refresh токена
    - Сохранение refresh токена в БД в данном проекте не реализовывалось
- **Создание, редактирование, удаление задач**
- **Фильтрация и поиск задач по статусу, приоритету**
- **Изменение статуса задачи**
- **Создание, редактирование, удаление групп**
- **Добавление и удаление задач из групп**

## Для администраторов:
- **Полный доступ ко всем задачам и пользователям**
- **Аналитика системы**
- Пользователь c admin1@mail.ru создан. Пароль - admin1 

## Установка и запуск:
- Приложение и БД размещены в Docker контейнерах
- **Разработка проводилась с использованием Docker Toolbox (из-за Windows 8)**
- **Для Docker Toolbox:** `http://192.168.99.100:8080/swagger-ui/index.html#/`
- **Для Docker Desktop:** `http://localhost:8080/swagger-ui/index.html#/`

## Аутентификация
- Новый пользователь по умолчанию получает роль `USER`
- Роль админа указывается при регистрации определенных email
    - В данном проекте список email-админов размещен в переменной окружения (хардкод)
    - Админы по умолчанию: `admin1@mail.ru`, `admin2@mail.ru`
- Пользователь с правами админа имеет две роли: `USER` и `ADMIN`

## Переменные окружения (установлены в `docker-compose.yml`):
- `ADMIN_EMAILS` - список email администраторов через запятую
- `JWT_SECRET` - секретный ключ для JWT
- `DATABASE_URL`, `DATABASE_USERNAME`, `DATABASE_PASSWORD` - параметры БД
